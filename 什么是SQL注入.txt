SQL注入其实就是改变SQL的结构，或者说让数据库误解的你意思。

比如：  select * from user where name = 'jack' and pwd = '' or 1=1 ;#'';   //   ' or 1=1 ;#'  这是用户传递给来的。

	以上那条语句就会与下面的一样，也就说 where 后面的都没有用了

	select * from user;

举个生活中的例子：
	本来你想要说：  我是男人，我爱女人。  然后男人，女人是用户传递的。

	但是用户传递了 ： 是不是男人，爱不爱女人  过来，那么整句话就不是我们想要的意思了

	我是不是男人，我爱不爱女人。


那么预处理又是为什么可以防SQL注入的。
	预处理先告诉数据库：  我是xx,我爱yy  这是固定的。一会传递过来的都是字符串或者数字。

	那么一会不管用户给数据库传递了什么？数据库都不会误解了。
	
	我是“不是男人”，我爱“不爱女人”
